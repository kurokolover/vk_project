# QuizHub Live: пояснительная записка

## 1. Цель проекта

QuizHub Live — работоспособный MVP веб-приложения для проведения квизов в реальном времени. Прототип закрывает основные сценарии: регистрация пользователей, разделение ролей на участника и организатора, создание квиза, запуск комнаты по коду, синхронная демонстрация вопросов, приём ответов только во время активного вопроса, подсчёт баллов, лидерборд и история проведённых/пройденных квизов.

## 2. Выбор средств разработки

Для клиентской части выбран React + Vite: это быстрый стек для прототипирования интерфейсов с хорошей скоростью сборки и понятной компонентной моделью. Для серверной части выбран Node.js + Express, так как требования проекта прямо предполагают JavaScript-сервер и удобное взаимодействие с клиентом. Для real-time сценариев используется Socket.IO: он упрощает работу с WebSocket-комнатами, переподключениями и событиями.

Для MVP данные сохраняются в файловое JSON-хранилище `backend/data/db.json`. Такой подход не требует отдельной настройки PostgreSQL или MongoDB, но модель данных спроектирована так, чтобы её можно было перенести в полноценную БД: пользователи, квизы, сессии, участники, ответы и результаты разделены явно.

## 3. Модель данных

Основные сущности:

- `users`: имя, email, роль, хеш пароля, дата создания.
- `quizzes`: организатор, название, описание, категории, правила, список вопросов.
- `questions`: тип вопроса `text` или `image`, режим выбора `single` или `multiple`, лимит времени, баллы, варианты ответов.
- `sessions`: активная комната с кодом, статусом, текущим вопросом, участниками, ответами и лидербордом.
- `participants`: пользователь, имя, счёт, количество верных ответов, дата подключения.

Пароли хешируются через `crypto.pbkdf2Sync`, авторизация реализована через подписанный токен. Для промышленной версии стоит заменить файловое хранилище на PostgreSQL, добавить миграции и refresh-токены.

## 4. Основные этапы разработки

1. Спроектирован интерфейс ролей: экран входа, кабинет, конструктор квиза, live-комната и история.
2. Реализован API для регистрации, входа, получения кабинета, создания и обновления квизов, запуска комнат.
3. Настроена Socket.IO-комната: подключение по коду, управление вопросами организатором, отправка ответов участниками, автоматическое закрытие вопроса по таймеру.
4. Добавлен подсчёт баллов: правильность ответа плюс опциональный бонус за скорость.
5. Реализован лидерборд и сохранение результатов в историю.
6. Собран адаптивный интерфейс с акцентом на рабочий сценарий, а не на лендинг.

Текущая структура разделяет проект на `frontend/`, `backend/` и `config/`. Frontend разбит по страницам, у каждой страницы есть свой JSX и CSS. Backend разделён на маршруты, auth-сервис, JSON-store, доменную логику квизов и realtime-слой Socket.IO. Docker-конфигурация находится в `backend/Dockerfile` и `docker-compose.yml`.

Локальный запуск:

```bash
npm install
npm run dev
```

Клиент открывается на `http://localhost:5173`, сервер работает на `http://localhost:3001`.
